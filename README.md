# RenewShield - 面向 TypeRenew 的轻量安全中心插件

本插件针对 TypeRenew 面向新手站长、轻量站点和虚拟主机的典型使用场景，提供了一套默认可用、易于管理的基础安全防护能力体系，有效帮助站点快速收窄攻击面。

**注意：仅用于基础安全防护，若站点已部署专业防护软件，或已接入 CDN、云 WAF 等服务，通常无需安装本插件。**

## 功能特性

- **基础请求防护**
  - 空 User-Agent 拦截
  - 脚本类 User-Agent 识别
  - 搜索引擎双向验证放行
  - 异常 HTTP 方法拦截
  - 基础 WAF 风险特征识别

- **AI 爬虫与自动化工具识别**
  - 主流 AI 爬虫标识识别
  - 自动化工具访问行为识别
  - 支持观察、验证、拦截、放行四种处理策略
  - 支持全局策略与单规则独立覆盖配置

- **浏览器一致性校验**
  - 浏览器最低版本限制
  - Sec-Fetch 头完整性校验
  - 浏览器基础请求头完整度检查
  - HTTP/1.x 协议风险识别
  - 代理转发头异常检测

- **访问挑战与频率限制**
  - 普通页面访问限频
  - 登录接口限频
  - 评论提交限频
  - XML-RPC 接口限频
  - 自动封禁与陷阱触发封禁

- **写请求与登录安全保护**
  - 写请求专项频率控制
  - 写请求重放攻击检测
  - 登录来源失败次数阈值控制
  - 撞库与密码喷洒攻击识别

- **搜索功能防滥用**
  - 搜索请求频率限制
  - 关键词枚举行为识别
  - 搜索词长度合法性检查

- **评论与上传安全**
  - 评论提交前置验证
  - 双扩展名上传拦截
  - 上传内容特征检测
  - 上传文件大小限制

- **访问控制与路径保护**
  - 基于路径或内容对象的访问规则配置
  - 支持登录态与用户角色权限控制
  - 支持 HTML 提示、403 拒绝、页面跳转三种处理方式
  - 支持指定路径启用 HTTP Basic Auth 验证

- **行为序列反爬虫**
  - 基于唯一路径访问数、搜索混合度与静态资源请求特征识别低速采集行为

- **运维管理面板**
  - 站点安全健康检查
  - 安全日志检索与筛选
  - 过期安全数据清理
  - 手动解除 IP 封禁

## 适用场景

- 基于 TypeRenew 构建的个人站点、博客、小型内容网站
- 运行在虚拟主机或轻量云服务器上的站点
- 需要基础安全防护，但不希望/暂无能力引入复杂安全系统的场景
- 希望在网站后台统一完成策略配置、日志查看与基础安全运维的站点

## 安装方式

1. 将插件目录上传至站点的 `/usr/plugins/` 目录
2. 确保目录名称为 `RenewShield`
3. 登录网站后台，进入"控制台 -> 插件"页面
4. 找到并启用 `RenewShield` 插件
5. 启用后，在左侧导航栏"插件"分组中点击"安全中心"进入配置界面

## 面板说明

插件后台面板分为以下功能模块：

- **总览**
  - 插件全局开关
  - 防护预设方案选择
  - 全局风险处理策略
  - WAF 规则处理策略
  - 缓存与日志保留时长设置

- **请求防护**
  - 基础请求防护配置
  - AI 爬虫与自动化工具策略
  - 浏览器一致性校验规则
  - IP 白名单、黑名单与扫描陷阱设置

- **挑战与限频**
  - 常规访问限频
  - 写请求保护
  - 登录安全保护
  - 搜索功能防滥用
  - 评论与上传安全
  - HTTP Basic Auth 与行为序列反爬

- **访问控制**
  - 受限访问规则管理
  - 默认拦截提示页 HTML 配置
  - 默认跳转地址设置

- **健康与日志**
  - 站点安全健康检查
  - 安全日志检索与筛选
  - 数据清理与 IP 解封操作

## 规则示例

### 访问控制规则

```text
/member/* => login => html
slug:vip => role:subscriber => redirect
cid:123 => role:editor|administrator => 403
```

### HTTP Basic Auth 路径规则

```text
/admin/*
/preview/*
/internal/tool
```

### AI 爬虫独立规则

```text
gptbot:challenge
claudebot:block
bytespider:observe
```

## 注意事项

- `RenewShield` 仅提供基础安全防护能力，不能替代专业安全设备或云 WAF 服务
- 启用更严格的防护规则前，建议先通过观察模式运行一段时间并结合日志评估影响
- 若站点部署在 CDN、反向代理或其他特殊网络环境后，请优先配置受信任代理列表
- 启用 HTTP Basic Auth 功能后，请同步配置对应的用户名、密码与受保护路径规则

# RenewShield

一款面向 TypeRenew 的轻量安全中心插件，适配 PHP 8 与 MySQL 8，提供请求防护、挑战限频、访问控制、健康检查与安全日志能力。

## 简介

`RenewShield` 的定位不是替代专业 WAF，也不是构建复杂笨重的安全体系，而是在 TypeRenew 面向新手站长、轻量站点和虚拟主机场景下，提供一套默认可用、易于管理、成本可控的基础防护能力。

插件保持 Typecho / TypeRenew 一贯的轻量思路：

- 尽量复用现有请求链路与插件钩子
- 不额外引入复杂服务依赖
- 不增加新的业务数据表
- 后台集中在“安全中心”面板管理
- 兼顾 PHP 8 与 MySQL 8 兼容性

## 功能特性

- **基础请求防护**
  - 空 UA 拦截
  - 脚本 UA 识别
  - 搜索引擎双向验证放行
  - 异常方法拦截
  - 基础 WAF 风险识别

- **AI 爬虫与自动化工具识别**
  - 支持主流 AI 爬虫识别
  - 支持自动化工具访问识别
  - 支持观察、验证、拦截、放行等策略
  - 支持按标识放行与按单独规则覆盖策略

- **浏览器一致性识别**
  - 浏览器最低版本要求
  - Sec-Fetch 校验
  - 浏览器基础头完整度检查
  - HTTP/1.x 风险识别
  - 代理头识别

- **挑战与限频**
  - 普通页面限频
  - 登录限频
  - 评论限频
  - XML-RPC 限频
  - 自动封禁与陷阱封禁

- **写请求与登录保护**
  - 写请求专项限频
  - 写请求重放检测
  - 登录来源失败阈值控制
  - 撞库喷洒识别

- **搜索防滥用**
  - 搜索请求限频
  - 关键词枚举识别
  - 搜索词长度检查

- **评论与上传保护**
  - 评论前验证
  - 双扩展上传拦截
  - 上传内容特征检测
  - 上传大小限制

- **访问控制与路径保护**
  - 基于路径或内容对象的访问规则
  - 支持登录态与角色控制
  - 支持 HTML、403、跳转三种处理方式
  - 支持 HTTP Basic Auth 路径验证

- **行为序列反爬**
  - 基于唯一路径数、搜索混合度与静态资源请求情况识别低速采集行为

- **运维面板**
  - 健康检查
  - 日志检索
  - 过期数据清理
  - 手动解除封禁

## 适用场景

- 使用 TypeRenew 的个人站、博客站、小型内容站
- 运行在虚拟主机或轻量服务器上的站点
- 需要基础安全防护，但不希望引入复杂安全系统的场景
- 需要在后台直接完成策略配置、日志查看与基础运维的站点

## 安装方式

1. 将插件目录放入站点的 `/usr/plugins/` 目录中
2. 确认目录名称为 `RenewShield`
3. 登录后台，进入“控制台 -> 插件”
4. 启用 `RenewShield`
5. 启用后，在左侧“插件”分组中进入“安全中心”进行配置

## 兼容性

- TypeRenew
- PHP 8.0+
- MySQL 8.0+
- PostgreSQL / SQLite

## 设计原则

- **轻量优先**：优先做基础防护，不堆叠复杂体系
- **最小改动**：尽量复用 Typecho / TypeRenew 现有能力
- **可观测**：所有关键动作尽量可在日志和健康检查中看到
- **可配置**：后台集中配置，无需手工改代码
- **兼容优先**：兼顾常见虚拟主机和共享环境

## 注意事项

- `RenewShield` 提供的是基础防护能力，不替代专业安全设备或云 WAF
- 启用更严格的规则前，建议先结合日志观察一段时间
- 若站点处于 CDN、反向代理或特殊网络环境，请优先配置受信代理
- 开启 HTTP Basic Auth 后，请同步配置用户名、密码与路径规则
